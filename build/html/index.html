Dokumentacja projektu portalu dla ludzi aktywnych¶

Wstęp¶

Portal dla ludzi aktywnych umożliwia umieszczenie trasy na mapie. Trasy są umieszczane i tworzone z wykorzystaniem Google Maps API. Do generowania wykresów wykorzystywane jest Google Chart API. Sam projekt został stworzony w języku Python i frameworku Django. Projekt korzysta z bazy danych sqlite3.

Niniejsza dokumentacja została stworzona przy pomocy narzędzia Sphinx.

Wszelkie zmiany w kodzie były umieszczane w zdalnym repozytorium Git w serwisie github.

Wymagania¶

  • system linuks
  • python 2.6.5
  • django 1.1.1
  • SQLite 3.6.20
  • przeglądarka z włączoną obsługą javascript oraz ciasteczek (ang. cookies)
  • dostęp do internetu

Instalacja¶

Projekt można pobrać bezpośrednio z repozytorium:

$ git clone git@github.com:mrplum/Mapy.git mapy
$ cd mapy
$ python manage.py syncdb
$ python manage.py runserver

Polecenie python manage.py syncdb tworzy potrzebne tabele. Polecenie python manage.py runserver uruchamia wbudowany serwer django.

Moduł map – maps¶

Moduł maps odpowiada za tworzenie oraz przeglądanie map. Przy tworzeniu map wykorzystywane jest API Google Maps napisane w JavaScript. Na mapach możliwe są następujace operacje:
  • tworzenie ścieżki
  • tworzenie punktów

Każda mapa którą utworzy użytkownik może zostać zapisana w bazie danych.

Model¶

class maps.models.Map(*args, **kwargs)¶

Każda mapa posiada nazwę, przyjazny adres utworzony na podstawie nazwy, słowa kluczowe, miasto, ciąg punktów z których złożona jest ścieżka, długość trasy w kilometrach oraz współrzędne krańcowe mapy.

Nazwa mapy jest ciągiem znaków nieprzekraczającym 256. Przyjazny adres (ang. slug) to ciąg znaków nieprzekraczający 50. Słowa kluczowe

getcenter()¶

Zwraca środek mapy w oparciu o współrzędne krańcowe trasy.

getlatlngs()¶

Zwraca tablicę par (lat, lng). Atrybut latlngs zawiera ciąg znaków odpowiadających kolejnym punktom na trasie.:

10.0,20.3;20.4,50.4;
Powyższy ciąg zostanie zmieniony w tablicę par.
>>> map.getlatlngs()
[(10.0, 20.3), (20.4, 50.4)]
gettags()¶

Każde słowo kluczowe jest ciągiem znaków. Atrybut tags zawiera słowa kluczowe oddzielone pojedynczym odstępem.

Metoda zwraca tablicę słów kluczowych.
>>> map.gettags()
['wroclaw', 'grunwaldzka', 'rondo-regana']
save()¶

Zapisuje mapę do bazy danych tworząc przyjazną nazwę mapy (ang. slug) używaną przy tworzeniu adresów map. Zamienia słowa kluczowe na wersję bez polskich znaków, spacji oraz znaków interpunkcyjnych.

Formularz¶

class maps.forms.NewMapForm(data=None, files=None, auto_id='id_%s', prefix=None, initial=None, error_class=<class 'django.forms.util.ErrorList'>, label_suffix=':', empty_permitted=False)¶

Klasa reprezentuje formularz umożliwiający dodawanie mapy do bazy danych. Atrybuty klasy:

  • name – nazwa mapy
  • tags – słowa kluczowe oddzielone przecinkiem
  • city – miasto lub miasta
  • latlngs – ciąg współrzędnych geograficznych oddzielonych średnikiem
  • distance – długość trasy
  • nelat
  • nelng
  • swlat
  • swlng

Wszystkie atrybuty są wymagane. Ukrytymi atrybutami są: nelat, nelng, swlat, swlng. Atrybut jest ukryty jeśli nie występuje w formularzu, do którego użytkownik podaje niezbędne dane

Widoki¶

maps.views.index(request)¶

Zwraca listę wszystkich map dostępnych w serwisie. Jeśli ustawiony jest parametr m to wtedy zwraca list map z miasta o wartości m. Jeśli ustawiony jest parametr tag to w liście pojawiają się mapy, które opisano słowem kluczowym tag.:

/mapy/?m=Wrocław
/mapy/?tag=sienkiewicza
/mapy/?m=Wrocław&tag=sienkiewicza
maps.views.like¶

Dodaje mapę do ulubionych zalogowanego użytkownika.

Parametry:
  • map_id – identyfikator mapy w bazie danych
  • slug – przyjazny adres mapy
Zwraca:

pusty obiekt HttpResponse

Funkcja jest wywoływana asynchronicznie przez metodę javascript.

Funkcja zwraca stronę z formularzem do dodania nowej mapy lub zapisuje mapę w zależności od typus żądania: GET (utwórz formularz) lub POST (zapisz formularz).

obiekt mapy, formularz komentarza, komentarze

Moduł profilów użytkownika – profiles¶

Profile użytkownika zawierają podstawowe informacje na temat aktywności danego użytkownika. Aktywność rozumiemy tutaj jako liczbę przebytych kilometrów, częstość przemierzania ulubionych tras.

Formularze¶

class profiles.forms.EditProfileForm(data=None, files=None, auto_id='id_%s', prefix=None, initial=None, error_class=<class 'django.forms.util.ErrorList'>, label_suffix=':', empty_permitted=False)¶

Reprezentuje formularz edycji danych. Edycja jest uzupełniona dodatkowo o first_name oraz last_name, które są nieobowiązkowe. Jeśli pole password nie zostało uzupełnione to hasło nie zostanie zmienione.

class profiles.forms.LoginForm(data=None, files=None, auto_id='id_%s', prefix=None, initial=None, error_class=<class 'django.forms.util.ErrorList'>, label_suffix=':', empty_permitted=False)¶

Reprezentuje formularz do logowania. Atrybuty username oraz password są obowiązkowe.

class profiles.forms.ProfileForm(data=None, files=None, auto_id='id_%s', prefix=None, initial=None, error_class=<class 'django.forms.util.ErrorList'>, label_suffix=':', empty_permitted=False)¶

Reprezentuje formularz tworzenia konta. Wszystkie atrybuty są obowiązkowe przy uzupełnianiu formularza:

  • username – nazwa użytkownika
  • password – hasło
  • email – adres e-mail

Moduł statystyk i wykresów – stats¶

Moduł obsługuje przetwarzanie wyników, pokonanych kilometrów oraz generowanie wykresów na podstawie zebranych danych.

Model¶

class stats.models.Time(*args, **kwargs)¶

Klasa reprezentuje rekord z wynikiem czasowym osoby, która przebyła pewną trasę. Atrybuty:

  • map – wskazuje obiekt mapy
  • user – wskazuje obiekt użytkownika
  • rounds – liczba rund
  • seconds – liczb sekund
  • distance – przebyty dystans
  • part_of_the_day – część dnia o której przebyto trasę
  • weather – pogoda
  • note – notatka
  • created_at – data utworzenia wpisu
save()¶

Oblicza dystans przebytej trasy i zapisuje rekord do bazy.

Uwaga

Jeśli rounds=10, a map.distance=10.5 km to przebyto 105.0 km, więc distance=105.0

static to_sec(template)¶

Zamienia szablon postaci gg:mm:ss lub mm:ss na sekundy

>>> Time.to_sec('10:23:44')
37424
>>> Time.to_sec('10:10')
70

Moduł ulubionych treści – favorites¶

Moduł umożliwia odznaczanie istniejących w bazie danych rekordów jako ulubionych dla określonych użytkowników. Moduł jest na tyle uniwersalny, że pozwala odznaczyć dowolny typ obiektu: komentarz, mapę, wypowiedź, zdjęcie etc. Wykorzystywany jest tutaj mechanizm tabel generycznych.

Modele¶

class favorites.models.Favorite(*args, **kwargs)¶
Reprezentuje model łączący ulubiony obiekt użytkownika z tym użytkownikiem.

  • content_type – typ obiektu
  • object_id – identyfikator obiektu
  • content_object – wskazuje na obiekt, który lubi użytkownik
